AJAX는 JavaScript의 라이브러리 중 하나이다.
AJAX는 자바스크립트를 사용하여 웹페이지를 리로드(전체 리소스 다시 다 불러야함)하지 않고, 
필요한 데이터만 서버에 요청하여 XML 또는 JSON 형태로 받아 페이지 갱신을 할 수 있다.
기존의 HTTP 프로토콜은 클라이언트에서 request를 보내고 서버에서 response를 받으면,
이어진 연결이 끊긴다. 때문에 페이지의 내용을 갱신하기 위해서는 다시 request를 보내고,
서버에서 response를 받아, 다시 페이지를 리로드 해야하므로 자원 낭비에 대한 불편함이 있었다.
그러나 AJAX는 XML의 XMLHttpRequest 메소드를 통해 서버에 request를 하면서, 필요한 데이터만
받아 페이지를 갱신할 수 있다.
AJAX의 기본 구동방식은 다음과 같다. 
AJAX는 웹 페이지의 디자인 요소와 정보 요소를 분리한다.
처음 페이지를 로드할 때, 해당 페이지의 스타일과 레이아웃을 정의하는 정적HTML 파일과
CSS 파일로 페이지의 기본적인 골격을 구축해 놓은 후, 
전달받은JavaScript(서버에 데이터를 어떻게 요청하면 되는지 설명한)파일로, 
서버에 데이터만을 별도로 요청한다. 이때 서버에서 받는 응답은 순수한 데이터 객체(XML, JSON)로,
웹브라우저가 이 데이터를 받게 되면, 앞서 만들어 놓은 틀 안에 적절한 방법으로 데이터를 끼워넣는다.

**AJAX의 단점 : 보안에 더 신경을 써야함, 오히려 연속으로 데이터 요청을 하게되면, 서버 부하가 증가함
**Jquery와 AJAX : jquery와 ajax를 함께 사용하면, HTTP GET방식과 POST 방식 모두 사용하여 원격 서버에 데이터 요청 가능
